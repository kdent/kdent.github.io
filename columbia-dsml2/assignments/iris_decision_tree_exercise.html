Exercise: Build a Decision Tree by Hand — Iris Dataset

The iris dataset contains measurements of three types of iris flowers:

  • Setosa
  • Versicolor
  • Virginica

Each flower has four measurements:

  • Sepal length
  • Sepal width
  • Petal length
  • Petal width

Your goal is to create a simple decision tree by hand that can classify the flowers.

Step 2: Choose a Decision Node

A decision tree starts with a question about one feature. Choose a feature and a cutoff value that separates one class from the others.

Examples:

  • Is petal length ≤ 2.5 cm?
  • Is petal width ≤ 0.8 cm?
  • Is sepal length ≤ 5.5 cm?

Your goal:

Find a rule that puts as many flowers of one class as possible into one branch.
Petal length ≤ 2.5? / \ Yes No ? ?

Step 3: Label the Branches

For each branch, decide:

  • Which class does this branch mostly contain?
  • Should the tree stop here?
  • Does another decision node need to be added?

Example:

Petal length ≤ 2.5? / \ Yes No Setosa Continue

Step 4: Improve Your Tree

Try adding another question to separate the remaining flowers.

Example:

Petal length ≤ 2.5? / \ Yes No Setosa Petal width ≤ 1.7? / \ Versicolor Virginica

Discussion Questions

  1. Which feature was most useful for splitting the data?
  2. Why do you think petal measurements might work better than sepal measurements?
  3. Could your tree make mistakes? Why?
  4. How would you write Python code to find the best split automatically?
